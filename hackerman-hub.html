▶ BUG BOUNTY WORKFLOW
The exact process real hunters follow. Do this in order every time.
1
READ THE SCOPE FIRST

Before touching anything — read the program brief on HackerOne / Bugcrowd. Know exactly what's in scope. Out-of-scope reports = instant reject + possible ban. Find programs →

2
PASSIVE RECON — DON'T TOUCH YET

OSINT only. whois, shodan.io, crt.sh/?q=%.target.com, Wayback Machine, GitHub search for leaked keys.

3
ACTIVE RECON — ENUMERATE

Now hit the servers. Subdomain enum with subfinder, alive check with httpx, port scan with nmap, dir fuzz with ffuf.

4
FINGERPRINT THE STACK

Figure out what's running. Wappalyzer, whatweb, check response headers, find API docs at /swagger or /graphql, read all JS files.

5
MANUAL TEST IN BURP SUITE

This is where the money is made. Intercept every request. Test every input. Create 2 accounts and test IDOR. Check every POST for CSRF. Try privilege escalation.

6
CONFIRM & DOCUMENT

Make sure it's real and reproducible. Screenshot everything. Save raw HTTP request/response. Note the exact vulnerable parameter.

7
WRITE A KILLER REPORT

Use the Report Builder tab. Clear title, severity, steps to reproduce, PoC, impact, fix suggestion. Report quality = bigger payout.

📖 VULN GUIDE
What each bug is, where to find it, how to test it. Learn these cold.
SQL INJECTION CRITICAL

Inject SQL into inputs to dump DBs, bypass auth, or RCE.

Where: Login forms, search, URL params with IDs.

Test: Add ' to any input — SQL error = potential hit.

Tool: sqlmap

Learn on PortSwigger →
XSS HIGH

Inject JS that runs in other users' browsers. Steal sessions/cookies.

Where: Any input that reflects in the page — search, comments, profiles.

Test: <script>alert(1)</script> in every field.

Tool: dalfox, XSStrike

Learn on PortSwigger →
IDOR HIGH — MOST COMMON

Access other users' data by changing an ID. Easiest money in BB.

Where: /api/users/1337 → try /api/users/1338

Test: 2 accounts. Access Account B's data from Account A's session.

Learn on PortSwigger →
CSRF MEDIUM

Trick a logged-in user's browser into making unauthorized requests.

Where: State-changing POSTs without CSRF tokens.

Test: In Burp, remove CSRF token — does it still work?

Learn on PortSwigger →
SSRF CRITICAL

Make the server fetch internal URLs. Exposes AWS metadata, cloud creds.

Where: Any URL input — webhooks, PDF generators, image importers.

Test: Try http://169.254.169.254/ as URL value.

Learn on PortSwigger →
SUBDOMAIN TAKEOVER HIGH

Dangling DNS pointing to deprovisioned cloud service you can claim.

Where: Subdomains returning "NoSuchBucket" or "There is no app here".

Tool: nuclei -t takeovers/

Easy money. Runs fast. Worth scanning every program.
